# CarbohydrateConfirmation
スイーツの栄養成分情報を取得するスクレイピングツール

## はじめに
私は、食事管理アプリ「あすけん※」を用いて、ダイエットを進めています。
あすけんは、朝昼夜+間食を規則し、エネルギーや三大栄養素、ビタミンの摂取量を数値で確認できます。また、食事は手入力だけでなく、画像解析でも記録できるなど手軽に管理できるアプリケーションです。
※https://www.asken.jp/

### 個人的な課題
あすけんを続ける中で、私は「夜ご飯食べ過ぎ問題」にぶつかりました。
朝昼は健康的な食事ができるのですが、一日の疲れかどうしても夜ご飯を食べ過ぎてしまい、胃腸に悪影響が生じてしまいました。

### 解決法
解決法として、夜ご飯前にスイーツを食べることで満足感を満たし、食べ過ぎを防ぎました。
しかし、用意するスイーツは、朝昼の栄養摂取量によって左右されて決めるのに時間がかかってしまいました。
そこで、朝昼の栄養摂取量をもとに夜に食べても問題ないスイーツを提案するシステムを作ろうと決意しました。

### 構成
- データの準備
	1. スイーツの栄養成分をスクレイピングで取得
	2. 取得したデータをcsvファイルに書き込む
- 本機能
	1. Discordへ摂取栄養表の画像をアップロード
	2. botが生成AIへ画像を送信
	3. 画像の内容を文字に起こして取得
	4. 取得した内容から摂取可能な栄養素の量を確認
	5. 摂取可能な栄養素の量をもとに食べても良いスイーツをcsvファイルから取得
	6. 取得した商品名をDiscordのチャットで表示

### まとめ
栄養成分を取得するスクレイピングツールは問題なく作成できました。
（なぜかファミマはスイーツだけ栄養成分を公開していなかった...）
